Every [release](https://github.com/sf666/nextcp2/releases) provides several ready-to-run artifacts, built automatically by CI. Pick the one that fits your platform — the native apps and the Docker image **bundle a Java runtime, so no separate Java installation is required**.

| Artifact | Platform | Java needed? |
| --- | --- | --- |
| `nextCP2-<version>.dmg` | macOS installer | no (JRE bundled) |
| `nextCP2-macos.zip` | macOS portable app image | no (JRE bundled) |
| `nextCP2-<version>.msi` | Windows installer | no (JRE bundled) |
| `nextCP2-windows.zip` | Windows portable app image | no (JRE bundled) |
| `nextcp2.jar` | any OS | yes (JRE 25) |
| `nextcp2-ma9000.jar` | any OS (optional driver) | — |
| `ghcr.io/sf666/nextcp2` | Docker (Linux) | no (in image) |

After starting, open `http://localhost:8085` in your browser (or `http://<host-ip>:8085` from another device).

## macOS

Install from the `.dmg`, or unzip the portable app image and move `nextCP2.app` where you like.

### Allow the app on first launch

The macOS app is **not notarized by Apple**, so on first launch macOS shows *"Apple could not verify that nextCP2.app is free of malware"* and refuses to open it. This is expected for a self-built, unsigned app — it does not mean anything is wrong.

To allow it, remove the download quarantine flag once:

```bash
sudo xattr -dr com.apple.quarantine /Applications/nextCP2.app
```

Adjust the path if the app is not in `/Applications` (e.g. `~/Downloads/nextCP2.app`). Afterwards the app opens normally on double-click.

Alternatively, open it via the UI: **System Settings → Privacy & Security**, scroll to the blocked-app notice and click **Open Anyway**, then confirm with Touch ID / password.

## Windows

Run the `.msi` installer (adds nextCP/2 to the Start menu), or unzip the portable app image and run `nextCP2.exe` from the extracted folder — no installation required.

Windows SmartScreen may warn that the publisher is unknown, because the app is not code-signed. Choose **More info → Run anyway** to continue.

## Runnable JAR

For servers, a NAS or a Raspberry Pi, the plain `nextcp2.jar` is usually the best fit. It needs a **JRE 25** on the machine:

```bash
java -jar nextcp2.jar
```

The optional `nextcp2-ma9000.jar` is the McIntosh MA9000/MA12000 device driver; place it in the configured device-driver library path (see [App Settings](/nextcp2/user_interface/app_settings/)).

## Docker

A Linux image is published to the GitHub Container Registry on every release:

```bash
docker pull ghcr.io/sf666/nextcp2:latest
```

See [Docker](/nextcp2/quick_install/docker/) for a complete `docker compose` setup (including the recommended UMS media server).
