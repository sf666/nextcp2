## Server Configuration

Displays a card view of the detected media server.

<Group>
  <Field name="Enable">Disabled servers will not be available for selection in certain pop-ups or lists.</Field>
  <Field name="Status">Previously discovered servers may appear as <code>OFFLINE</code> if not yet discovered on the network; active servers are <code>ONLINE</code>. Read only.</Field>
  <Field name="IP">IP address of the device. Read only.</Field>
  <Field name="UDN">Unique Device Name, identifies the device on the network. Read only.</Field>
  <Field name="MY PLAYLIST — ObjectID">Media server ObjectID of a folder containing playlists. Usually configured automatically by the UI; once set, the playlists from that folder appear in the sidebar.<Note>If the server's database is recreated or reindexed, the ObjectID may change and has to be set again.</Note></Field>
  <Field name="ID3 Rating">If checked, enables reading and writing star ratings to audio file metadata, if the format supports it.<Note>Only available for the UMS media server (fork).</Note></Field>
</Group>

## Renderer configuration

Displays a card view of the detected media renderers. Offline renderers may appear if not yet discovered; disabled renderers are excluded from selection lists.

#### Device driver

If the renderer is connected to a physical receiver or AV device with remote-control capability (e.g. via RS232 or TCP/IP), a device driver can control volume and balance directly. A driver for McIntosh devices is available for download.

<Group>
  <Field name="amplifier type">Select the device driver, e.g. <code>MA9000</code> for McIntosh amplifiers.</Field>
  <Field name="amplifier driver connection">Driver connection string. For the <code>MA9000</code> driver, e.g. <code>192.168.112.37:12345</code>.</Field>
  <Field name="amplifier on power default volume">Restores the amplifier to a predefined volume when powered on, e.g. <code>10</code>.</Field>
  <Field name="amplifier default balance">Restores the amplifier to a predefined balance when powered on, e.g. <code>-3</code>.</Field>
  <Field name="UPnP/streamer max volume">If checked, sets the UPnP/streamer to max volume (100%) when powered on.</Field>
</Group>

## Server Actions

Additional UMS server actions, available when the active media server is a UMS instance.

<Group>
  <Field name="rescan media library">Rescans the UMS media library. Can be resource-intensive on large libraries.</Field>
  <Field name="backup liked albums">Backs up UMS liked albums to a separate file on the UMS server, so they can be restored if the library is deleted.</Field>
  <Field name="restore liked albums">Restores liked albums from a backup file, if one exists.</Field>
  <Field name="import rating from musicBrainz.org">Imports star ratings from the musicbrainz.org account configured under Accounts.</Field>
</Group>

## App Behavior

<Group>
  <Field name="Reload page on focus">When enabled, returning to the tab/window reloads the current page. This is a browser-local preference (stored per browser), not part of the server configuration.</Field>
</Group>

## General Configuration

<Group>
  <Field name="device driver library path">Where to look for amplifier device drivers. McIntosh amplifiers such as MA9000 / MA12000 (and similar) should work.</Field>
  <Field name="database path">Where nextCP/2 stores its internal settings.</Field>
  <Field name="unsecure server listening port">The HTTP port you connect to with your browser.</Field>
  <Field name="path to restart script">Full path to a restart script; read access is required.<Note>Experimental — may not work.</Note></Field>
  <Field name="Log4j config file">Where to look for a Log4j2 configuration file.</Field>
  <Field name="Behind reverse proxy">Trust the <code>X-Forwarded-*</code> headers so nextCP/2 reconstructs the correct external scheme and host when TLS terminates at a proxy. Enable only when nextCP/2 is reachable solely through a trusted proxy — see <a href="/nextcp2/quick_install/docker/#running-behind-a-reverse-proxy-traefik">running behind a reverse proxy</a>. Default: off.</Field>
  <Field name="SSE heartbeat interval">Interval in seconds for the Server-Sent-Events keep-alive ping that stops proxies from dropping the live <code>/SSE</code> and <code>/mcp</code> streams. Set <code>0</code> to disable. Default: 30.</Field>
</Group>

<Group title="SSL">
  <Field name="secure server listening port">The HTTPS listening port. Works only if a P12 keystore is configured below.</Field>
  <Field name="P12 keyfile file location">Path to the nextCP/2 keystore for secure connections. Must be in P12 format.</Field>
  <Field name="P12 keyfile password">Password for the P12 keystore.</Field>
</Group>

<Group title="UPnP">
  <Field name="Generate UPnP code">If <code>true</code>, <code>JUPnP</code> access wrappers are generated for discovered UPnP services.</Field>
  <Field name="bind interface stream server">Leave empty to bind to all interfaces. Changes require a restart.<Note>If nextCP/2 runs inside a Docker container, select the host interface here.</Note></Field>
  <Field name="path for generated UPnP code">Folder where generated UPnP code is saved.</Field>
</Group>

<Group title="Browser Player">
  <Field name="transcode before streaming">Applies to the built-in browser player ("This device"). If checked (default), a track that needs transcoding is fully transcoded before playback starts, enabling correct duration display and seeking. Uncheck for lower latency, at the cost of seeking on transcoded tracks.</Field>
  <Field name="cache directory">Directory where transcoded audio for the browser player is cached. Leave empty to use the system temp directory.</Field>
  <Field name="cache max size">Maximum size of the transcode cache in MB. Set <code>0</code> for unlimited.</Field>
  <Field name="cache idle TTL">How long (in hours) a cached transcode is kept after its last use. Set <code>0</code> to keep entries until the next restart.</Field>
</Group>

## AI Configuration

Configures the built-in [AI chat assistant](/nextcp2/ai/chat_assistant/). Changes are applied immediately after saving — no restart is required. For provider setup examples (Google Gemini, OpenWebUI) and troubleshooting, see [AI Configuration](/nextcp2/ai/ai_configuration/).

:::caution
The AI feature is experimental and under active development — these settings may change with any release.
:::

<Group>
  <Field name="AI enabled">Master switch for the AI assistant feature.</Field>
  <Field name="AI provider">Selects the active provider: <code>google</code> (Gemini) or <code>openai</code> (OpenAI and OpenAI-compatible endpoints like OpenWebUI). Switching providers saves the current settings to a per-provider profile and restores the settings last used with the newly selected provider.</Field>
  <Field name="Base URL">Only for OpenAI-compatible endpoints; disabled for Google. Enter the API root, e.g. <code>http://localhost:3000/api</code> for OpenWebUI.<Note>Do not append <code>/v1</code> or <code>/chat/completions</code> — these are added automatically.</Note></Field>
  <Field name="API key">Provider API key. Required for Google; for OpenAI-compatible endpoints it depends on the server.</Field>
  <Field name="Model">The model to use. The list is fetched live from the provider and can be filtered by typing.</Field>
  <Field name="Send nextCP tools">If checked (default), nextCP/2 sends its built-in tools (device selection, radio playback, language switching) to the model. Uncheck if your endpoint provides its own server-side tools.</Field>
  <Field name="Conversation memory">If checked, recent messages of the conversation are sent as context with each request, so follow-up questions work. Increases token usage. Default: off.</Field>
  <Field name="Tool IDs">OpenWebUI only: comma-separated list of server-side tool IDs to attach to each request, or <code>*</code> to use all tools registered on the endpoint.</Field>
  <Field name="Server-side tools">OpenWebUI only: if the endpoint exposes tools, they are listed as checkboxes for granular selection.</Field>
</Group>

## Accounts

#### Spotify

Connect your Spotify account to enable discovery of artist album art. If the browser is connected to localhost, a one-step linking process is available. Otherwise the <code>authCode</code> must be copied manually into the provided code block.

#### Last.fm

The account can be connected, but the API is not used yet.
